# 2023 电子设计竞赛 E 题实施计划

## 总目标

完成 2023 年全国大学生电子设计竞赛 E 题“运动目标控制与自动追踪系统”，并优先满足以下目标：

- 基本要求全部完成
- 发挥部分的追踪过程尽量稳定
- 视觉链路尽量稳定在 `30 fps`
- 系统结构满足“红色系统与绿色系统相互独立、不可通信”的要求

## 总体路线

系统建议拆分为两个彼此独立的子系统：

1. 红色激光点运动控制子系统
2. 绿色激光点自动追踪子系统

推荐原则如下：

- 红色子系统尽量采用“轨迹生成 + 标定 + 控制”
- 绿色子系统尽量采用“固定相机 + 传统视觉 + 闭环追踪”
- 不优先使用深度学习
- 不优先上 CUDA
- 先用 OpenCV 跑通，再考虑局部性能优化

## 第一阶段：需求拆解

先把题目要求拆成可实现的功能块：

- 红色激光点回到原点
- 红色激光点沿 `0.5m` 方框运动一周
- 红色激光点沿 A4 黑胶带边框运动一周
- A4 纸任意旋转摆放后，仍能沿边框运动一周
- 绿色激光点在 `2` 秒内追上红色激光点
- 当红点运动时，绿点能够持续稳定跟踪
- 暂停后可以直接测量两点间距离

## 第二阶段：硬件架构

### 红色子系统

- MCU 或其他嵌入式控制器
- 二维云台
- 红色激光器
- 可选本地相机，用于目标纸张识别与标定

### 绿色子系统

- Jetson Nano
- 固定朝向屏幕的 USB 摄像头
- 二维云台
- 绿色激光器
- 蜂鸣器或 LED，用于提示追踪成功

### 关键约束

两个子系统不能通信，因此应避免：

- 共享控制器
- 共享串口、总线或无线链路
- 共享同一套核心处理器和相机资源

最稳妥的做法，是让两套系统在硬件和控制链路上都保持独立。

## 第三阶段：机械与光学布置

- 固定白色屏幕
- 固定红色激光器与红色云台
- 将绿色激光器布置在允许的运动范围内
- 固定绿色系统摄像头，确保完整覆盖屏幕区域
- 确保相机安装后不晃动
- 尽量为镜头和激光提供遮光条件，减小环境光干扰

## 第四阶段：标定

### 屏幕标定

- 确定屏幕四角和中心位置
- 建立像素坐标到屏幕物理坐标的映射
- 推荐使用四点透视变换

### 云台标定

- 标定“控制量 -> 屏幕坐标”的映射关系
- 至少需要建立：
  - 红色云台控制量到红点落点的映射
  - 绿色云台控制量到绿点落点的映射

推荐方法：

- 使用网格采样
- 结合双线性插值或低阶多项式拟合

## 第五阶段：红色子系统方案

### 5.1 回到原点

- 通过标定表直接得到原点对应的控制量
- 再进行小范围闭环微调

### 5.2 `0.5m` 方框轨迹

- 固定方框位置
- 在物理坐标系中生成四条边上的轨迹点
- 让红色云台按轨迹点依次运动

### 5.3 A4 黑胶带边框

- 通过视觉方法一次性检测黑胶带矩形
- 提取四个角点
- 建立矩形边线参数
- 生成顺时针或逆时针的轨迹点

### 5.4 任意旋转的 A4 纸

- 同样通过矩形检测得到旋转角度
- 直接根据四角点生成旋转矩形轨迹

### 5.5 红色子系统的视觉方法

如果红色子系统也需要视觉辅助，建议采用：

- 灰度化
- 阈值分割黑胶带区域
- `findContours`
- 选择最大且近似四边形的轮廓
- 提取四个角点

这里的视觉任务不是逐帧跟踪，而是“一次识别 + 轨迹生成”。

## 第六阶段：绿色子系统方案

### 6.1 相机流水线

- 目标帧率：`30 fps`
- 初始输入建议从 `640x480` 开始
- 处理阶段可缩放到 `320x240`
- 锁定目标后切换到 ROI

### 6.2 光斑检测

每一帧都执行以下步骤：

- 读取图像
- 计算红点得分：`2R - G - B`
- 计算绿点得分：`2G - R - B`
- 阈值分割
- 连通域或轮廓筛选
- 计算红点与绿点质心

### 6.3 闭环控制

- 误差定义：`e = red_center - green_center`
- 控制器：双轴 PID
- 输出目标：绿色云台控制量

### 6.4 锁定与跟踪逻辑

- 初始阶段：全图搜索红点和绿点
- 追上之后：只在上一帧附近开启 ROI
- 如果 ROI 中丢失目标：退回全图搜索

### 6.5 成功与失败判定

- 两点距离 `<= 3cm`：判定为追踪成功
- 超出阈值并持续累计：记为跟踪失败次数
- 连续稳定成功后触发声光提示

## 第七阶段：软件结构

建议程序按以下模块组织：

1. camera input
2. calibration manager
3. spot detector
4. target estimator
5. PID controller
6. gimbal driver
7. state machine
8. UI / debug overlay
9. logger

## 第八阶段：性能策略

为了尽量稳定达到 `30 fps`，优先做这些优化：

- 先用 OpenCV，不急着写 CUDA
- 关闭自动曝光、自动白平衡和自动对焦
- 把分辨率降到“够用即可”
- 进入跟踪阶段后使用 ROI
- 避免全图复杂滤波
- 尽量减少内存拷贝
- 如果要做最终比赛版本，优先考虑 C++ OpenCV

## 第九阶段：测试计划

建议按以下顺序逐项验证：

1. 摄像头能稳定输出 `30 fps`
2. 红色和绿色激光点能被稳定检测
3. 单独测试绿色云台跟踪静止红点
4. 单独测试红色轨迹运动
5. 测试绿色光点能否在 `2` 秒内追上静止红点
6. 测试绿色光点对运动红点的持续跟踪
7. 测试暂停与距离测量功能
8. 测试不同环境光条件
9. 测试不同绿色激光器安装位置

## 第十阶段：得分策略

推荐按以下顺序推进：

1. 先确保红色系统完成基本要求
2. 再单独做好绿色系统追静止点
3. 最后完善绿色系统跟踪运动红点

原因是：

- 红色系统是发挥部分的前提
- 红色轨迹稳定后，绿色系统调试会容易很多

## 当前最值得先做的事

建议下一步按这个顺序推进：

1. 确定最终硬件分工
2. 确定绿色系统固定相机的位置
3. 先编写绿色系统的红绿光斑检测与显示程序
4. 再完成绿色光点的闭环追踪
5. 最后编写红色轨迹生成与控制模块
