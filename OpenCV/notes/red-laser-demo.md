- 白色背景纸
- 黑色胶布
- 红色激光
- 绿色激光

使用方式：

1. 将目标放到画面中央的 `Sample ROI` 框内
2. 点击对应按钮
3. 程序会记录该区域的：

   - 亮点热点像素统计

4. 采样累计结果会显示在页面下方，也可通过 `calibration.json` 读取

## 稳健统计滤坏点

为了处理“点错按钮”或“采样框没有完全对准目标”的情况，标定摘要现在会对样本做两级过滤：

1. 先做物理约束预筛
   - 红激光要求：红色对抗得分为正、绿色对抗得分为负、`LAB a*` 偏红
   - 绿激光要求：绿色对抗得分为正、红色对抗得分为负、`LAB a*` 偏绿
   - 白纸和黑胶布按亮度范围做初筛
2. 再对关键特征做 `MAD` 稳健去异常
   - 返回 `accepted_counts`
   - 返回 `rejected_counts`
   - 返回 `rejected_ids`

这样在多次采样后，可以自动滤掉明显不合理的坏点，而不是直接用所有样本做均值。

## 检测方法

- 使用红色得分公式：`2R - G - B`
- 对红色得分和红色占优程度进行阈值判断
- 对候选区域做形态学膨胀，扩大激光点外围红色区域
- 选择“峰值红色得分最高”的候选区域，而不是只看平均得分
- 使用原始红色候选像素的加权质心输出位置

## 白背景红点修复

在白纸背景上，激光点中心往往会过曝，表现为“发白/发粉”，而不是纯红。
因此原先较高的阈值会把红点漏掉。

已做如下调整：

- `score_threshold`：`120 -> 30`
- `min_red`：`150 -> 120`
- `min_delta`：`40 -> 15`
- `min_area`：`3 -> 1`
- `max_area`：`400 -> 800`

修复思路：

- 不再只识别“纯红中心”
- 改为识别“红色边缘 + 发白中心”组成的激光光斑簇
- 用峰值选择主光斑，减少选错次级反射点的概率

## 关于 LAB

结论：

- `LAB` 可以提高算法设计的合理性
- 但不建议完全用 `LAB` 替换当前方法
- 更推荐把 `LAB` 当作辅助特征，尤其是 `a*` 通道

原因：

- `a*` 通道本质上描述红绿对立信息
- 白纸背景虽然亮，但通常 `a*` 接近中性
- 红色激光边缘在 `a*` 上往往比在原始 RGB 上更容易和背景分开

局限：

- 如果激光中心已经过曝成接近纯白，颜色信息本身就丢失了
- 这时不管是 RGB、HSV 还是 LAB，都无法从“纯白中心”里恢复真正红色

因此更合理的做法是：

- 用 `LAB` 的 `a*` 通道增强红色候选
- 同时保留亮点、小面积、峰值、邻域结构这些约束

推荐的混合思路：

1. 转到 `LAB`
2. 取 `a*` 通道作为红色主响应
3. 做局部对比增强，例如：
   - `a_local = a - blur(a)`
4. 再结合以下条件筛选：
   - `L` 足够亮
   - 候选区域面积足够小
   - 峰值明显
   - 保留当前的 RGB 红色占优条件作为辅助判别

对当前这个项目来说，最稳妥的方向不是“RGB 或 LAB 二选一”，而是：

- `RGB + LAB(a*) + 面积/亮度/峰值约束`

- `accepted_counts.red_laser = 12`
- `rejected_counts.red_laser = 0`
- 说明这组红激光样本内部一致，没有明显坏点

从这组样本里提炼出的关键结论是：

- `score_threshold` 可以降到接近 `4`
- `R-G` 的下限可以放宽到接近 `4`

- `min_delta_rg`
- `min_delta_rb`

并把默认运行阈值调整到更适合白背景激光点的范围：

- `score_threshold = 6`
- `min_red = 170`
- `min_delta_rg = 4`
- `min_delta_rb = 0`

- `accepted_counts.black_tape = 21`
- `rejected_counts.black_tape = 0`
- 说明这组黑胶布样本内部一致，没有明显坏点

- `ROI LAB L` 中位值约为 `37.51`
- `ROI HSV V` 中位值约为 `45.81`

这意味着后续黑框检测应以“区域足够暗”为主，而不是看热点颜色。
黑胶布样本里的热点像素主要反映环境反光和边缘高亮，不适合作为黑胶布主阈值。

对黑胶布的检测建议：

- 主特征优先使用 `LAB L`
- `HSV V` 作为辅助约束
- 后续在白纸样本采完后，再根据黑白分离间隔确定最终上限阈值

在白纸样本还没加入之前，可以先把黑胶布理解为：

- `LAB L` 明显低于白纸背景
- `HSV V` 明显低于白纸背景

## 2026-04-18 黑白分离结论

当前白纸和黑胶布样本已经足够说明，黑框检测应当走“亮度分割 + 几何约束”，而不是热点颜色分析。

现有中位值：

- 白纸 `ROI LAB L` 约为 `131.24`
- 黑胶布 `ROI LAB L` 约为 `37.51`

两者的亮度间隔约为 `93.73`，分离非常明显。

因此后续识别黑色矩形时：

- 主要看 `LAB L` 或灰度
- 用整块区域阈值分割
- 再用轮廓、面积、四边形拟合来锁定矩形

这类任务里，少量反光不会改变“整体是黑色矩形”的事实，因为：

- 反光只会影响局部像素
- 黑框本体仍然是大面积连续暗区域
- 最终判定依赖的是区域连通性和矩形几何，不是单个亮点

在当前这组样本下，固定阈值可先从以下范围起步：

- `LAB L < 80~90` 作为黑色候选区域

更稳的工程实现建议：

1. 先限定在白色背景纸或屏幕 ROI 内处理
2. 转 `LAB`，取 `L` 通道
3. 做轻微平滑
4. 以固定阈值或 `Otsu` 分割暗区域
5. 做形态学闭运算，补齐边缘缺口
6. 提取最大候选轮廓并做四边形拟合

## 2026-04-18 绿激光样本结论

一次 12 组绿激光采样中：

- `accepted_counts.green_laser = 10`
- `rejected_counts.green_laser = 2`
- 被拒样本 `id = 44, 51`

这说明稳健过滤已经开始发挥作用，能滤掉少量误操作或异常角度样本。

从有效样本看，绿激光的特点是：

- `hotspot_green_channel_p10` 仍然很高
- `LAB a*` 明显低于中性值 `128`
- 但在过曝中心处，`green_minus_red` 和 `green_minus_blue` 不一定始终保持明显正值

这也是“绿色偶尔被误判为红色”的根本原因：

- 白背景上激光中心会发白
- 如果只看单一路径的 RGB 阈值，局部像素可能短暂落入另一类激光的阈值区间

因此后续红绿区分的核心策略应改为：

- 红激光：`RGB 红色占优 + LAB a* 偏红`
- 绿激光：`RGB 绿色亮度占优 + LAB a* 偏绿`

也就是说，不再只用单一颜色阈值，而是：

- 同时做红、绿两路检测
- 用 `LAB a*` 作为颜色类别约束
- 避免绿色激光被红色检测器误收

## 当前黑框检测实现

已经把黑框检测算法加入到实时演示程序中，当前实现路线是：

1. 每隔几帧做一次黑框检测，而不是每帧重跑
2. 在缩小后的图像上取 `LAB L` 通道
3. 用固定阈值提取暗区域
4. 做闭运算和轻微膨胀，连通黑胶布边缘
5. 提取最大候选四边形
6. 要求候选框内部亮度足够高，以贴合“黑边框包围白底”的题目场景

当前默认参数：

- `black_detect_interval = 4`
- `black_detect_scale = 0.5`
- `black_l_threshold = 85`
- `black_inner_l_min = 105`

当前状态接口和叠加层中，已经新增：

- `black_frame.detected`
- `black_frame.area`
- `black_frame.corners`

说明：

- 这套算法已经能在画面中给出黑框候选四边形
- 但在没有实际题目靶纸、或者场景里存在其他“大暗边包亮内部”结构时，仍可能锁定到错误候选
- 如果后面把真实黑胶布矩形放到画面里，我们再根据现场图像继续收紧 ROI、面积范围和矩形约束即可

## 2026-04-18 黑胶布中线方案

在实际联调中，直接从黑框外边界和白内页边界同时求“真中线”会受光照和视角影响，容易在不同白色区域之间跳变。

最后采用的稳定方案是：

1. 先稳定检测黑胶布外边界四边形
2. 再把这个四边形按固定比例向内缩放
3. 缩放后的四边形作为“黑胶布粗线中线”的近似

这样做的优点是：

- 比直接取最外边缘更贴近黑胶布中线
- 明显比实时找内白框更稳
- 算法更轻，帧率可回到约 `30 fps`

当前关键实现点：

- 黑框检测加入长宽比约束，避免锁到顶部横向亮带
- 加入位置连续性约束，降低候选在多个大矩形之间跳变的概率
- 激光检测限制在黑框附近 ROI 内进行，减少全图搜索开销
- 黑框中线默认由外框按固定比例 `0.94` 向内缩放得到

## 2026-04-18 内外围取中线实验

后来又切回了一版更几何化的实现：

1. 先检测黑胶布外边界四边形
2. 对外边界做透视归一化
3. 在归一化图里提取白色内框
4. 将内框反投影回原图
5. 对内外两组角点取平均，得到中线四边形

这版代码已经部署到 Jetson 的实时检测脚本中。

当前观察到的结论是：

- 方案本身可运行
- 当黑框完整、居中、透视不过分时，`inner_corners` 能给出内框
- 当画面只拍到黑框的一部分，或者白色背景纸大面积进入视野时，内框会偶尔跳到不希望的白色区域

另外，Jetson 在 `systemd-run` 下运行这版脚本会触发 `Illegal Instruction`，但直接用 `jetson` 用户后台启动是正常的。
因此当前实时服务是用后台常驻进程拉起的，而不是 transient systemd unit。
